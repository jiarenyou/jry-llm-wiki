# 工厂主数据管理

主数据是工厂数据体系的"锚点"——所有业务数据、分析数据都需要通过主数据编码进行关联和解读。没有规范的主数据管理，[[工厂数据仓库建模]]中的维度表就无法建立，[[数据血缘与元数据管理]]中的数据链路也会断裂。主数据管理（Master Data Management, MDM）是工厂[[工业数据治理框架]]中最基础也最关键的环节。

## 主数据类型体系

工厂主数据涵盖五大类型，每一类都有独特的管理要求：

| 类型 | 核心实体 | 管理系统 | 变更频率 |
|------|----------|----------|----------|
| 设备主数据 | 设备、产线、工装夹具 | CMMS + MES | 低 |
| 物料主数据 | 原材料、半成品、成品、BOM | ERP + PLM | 中 |
| 工艺主数据 | 工艺路线、工序、参数标准 | MES + PLM | 中 |
| 人员主数据 | 操作工、工程师、班组 | HR + MES | 低 |
| 组织主数据 | 工厂、车间、产线、工位 | ERP + MES | 低 |

## 设备主数据

设备是工厂最重要的资产和分析对象，设备主数据的质量直接影响[[OEE-设备综合效率]]等核心指标的计算准确性。

### 核心字段

- **设备编码**：全厂唯一的设备标识，推荐编码规则 `工厂代码-车间代码-设备类型-序列号`（如 `SH-WS-CNC-001`）
- **设备类型**：CNC、注塑机、冲压机、装配线、检测设备等，关联设备类型标准库
- **规格参数**：额定功率、最大转速、工作行程等静态参数
- **所属产线**：设备与产线的归属关系，一个产线包含多台设备
- **供应商信息**：供应商名称、联系方式、保修期限
- **投用日期**：设备启用时间，用于折旧计算和维护计划

### 设备分类体系

建立设备分类树，支撑统计分析：
```
设备总类
├── 生产设备
│   ├── 机加工设备（CNC、车床、铣床）
│   ├── 成型设备（注塑、冲压、压铸）
│   └── 装配设备（自动装配线、机器人）
├── 检测设备
│   ├── 三坐标测量机
│   ├── 视觉检测设备
│   └── 功能测试设备
└── 辅助设备
    ├── 空压机
    ├── 冷却系统
    └── 输送系统
```

### 设备与数据采集点映射

每台设备关联多个数据采集点（传感器、PLC地址），需要在主数据中维护映射关系：
- 设备编码 → 采集点位列表（OPC UA Node ID / MQTT Topic）
- 每个采集点的语义定义（温度、压力、振动、计数等）
- 采集频率和数据类型

## 物料主数据

物料主数据是连接生产计划、物料管理和成本核算的关键纽带。

### 核心字段

- **物料编码**：全局唯一标识，推荐采用"分类码+特征码+序列号"的结构化编码
- **物料名称与描述**：标准化命名，避免"同物不同名"
- **物料分类**：原材料、辅助材料、半成品、成品、备品备件
- **计量单位**：基本单位（个/千克/米）及换算关系
- **BOM关系**：父子物料关系，定义产品的物料组成结构
- **存储条件**：温度要求、湿度要求、保质期、安全库存

### BOM管理

BOM（Bill of Materials）是物料主数据中最复杂的部分：
- **工程BOM（EBOM）**：由[[PLM-产品生命周期管理]]系统管理，反映产品设计结构
- **制造BOM（MBOM）**：由[[MES-制造执行系统]]使用，反映实际制造过程
- **成本BOM（CBOM）**：由[[ERP-企业资源计划]]使用，用于成本核算
- 三种BOM之间需要建立映射和转换规则

## 工艺主数据

工艺主数据定义了"产品应该怎么做"，是连接设计意图和生产执行的桥梁。

### 核心内容

- **工艺路线**：产品从原材料到成品经过的工序序列
- **工序定义**：每个工序的标准操作内容、所需设备和工具
- **参数标准**：每个工序的关键工艺参数及其标准值和公差范围
- **工时定额**：每道工序的标准工时，用于产能计算和成本核算
- **质量标准**：每道工序的检验项目、检验方法和判定标准

### 与SPC的关系

工艺主数据中的参数标准值和公差范围，是[[SPC-统计过程控制]]的控制限设定依据。当工艺参数标准变更时，SPC控制限需要同步更新——这正是[[数据血缘与元数据管理]]需要追踪的典型链路。

## 人员与组织主数据

### 人员主数据

- 人员编码、姓名、技能资质（可操作的设备类型）
- 班组归属、班次偏好
- 资质有效期管理（特种作业证等）
- 与[[MES-制造执行系统]]的权限控制关联

### 组织主数据

- 工厂层级结构：集团 → 事业部 → 工厂 → 车间 → 产线 → 工位
- 成本中心映射：每个组织单元对应的成本中心编码
- 组织变更的生效时间管理

## 主数据治理

### 编码规范

制定全厂统一的主数据编码规范是治理的第一步：

1. **唯一性**：每个实体有且仅有一个编码，不允许重复
2. **结构化**：编码中包含分类信息，便于识别和统计
3. **稳定性**：编码一旦分配不再变更（即使实体属性变化）
4. **可扩展性**：预留编码空间，支持新设备/物料类型的扩展

### 唯一性校验

在主数据创建和维护时，必须进行唯一性校验：
- 新建设备编码时，检查是否与已有编码冲突
- 新建物料编码时，检查是否已存在相同规格的物料
- 跨系统唯一性检查：ERP中的设备编码与MES中的设备编码必须一致

### 变更管理

主数据变更需要走审批流程：
- **变更申请**：说明变更原因和影响范围
- **影响分析**：评估变更对[[工厂数据仓库建模]]、报表、看板的影响
- **审批执行**：经相关部门审批后执行变更
- **通知同步**：变更结果同步到所有消费系统
- **版本记录**：保留变更历史，支持回溯

## 系统间的主数据协同

工厂中多个系统都涉及主数据管理，需要明确"单一数据源"（Single Source of Truth）：

| 主数据类型 | 主数据源（创建/维护） | 消费系统 |
|------------|----------------------|----------|
| 设备 | CMMS（或MDM平台） | MES、SCADA、数仓 |
| 物料 | ERP（或MDM平台） | MES、WMS、PLM、数仓 |
| 工艺 | PLM/MES | MES、数仓 |
| 人员 | HR系统 | MES、ERP |
| 组织 | ERP | MES、数仓 |

### 同步机制

- **实时同步**：关键主数据变更通过消息队列实时推送到消费系统
- **定时同步**：每日全量对比，发现差异告警
- **主数据中台**：建设统一的主数据管理平台，所有系统通过API读写主数据

## 实施建议

1. **先理后治**：先梳理现有主数据现状，识别问题（重复、缺失、不一致），再制定治理方案
2. **从设备开始**：设备主数据影响面广但复杂度适中，适合作为MDM建设的切入点
3. **工具辅助**：使用专业的MDM平台或自研主数据管理系统，避免Excel管理
4. **流程保障**：技术方案必须配合管理制度，确保主数据的"入口"受控

主数据管理是工厂数字化的基石。只有建立在准确、一致、完整的主数据之上，[[工厂数据仓库建模]]中的分析结果才具有可信度，[[智能制造概述与工业4.0]]的愿景才能真正落地。
