# MTTR — 平均修复时间

## 定义

MTTR（Mean Time To Repair）衡量的是**设备从故障发生到恢复正常运行所需的平均时间**。它反映的是维护团队的响应速度和修复能力。

## 计算公式

$$MTTR = \frac{\sum 故障修复时间}{故障次数}$$

## MTTR 的构成

MTTR 不仅是"修设备"的时间，而是从故障到恢复的全链路：

| 阶段 | 内容 | 占比（典型） |
|------|------|-------------|
| **发现** | 故障检测 + 报警 | 5% |
| **响应** | 维修人员到场 | 15-20% |
| **诊断** | 定位故障原因 | 20-30% |
| **修复** | 更换零件/调整参数 | 30-40% |
| **验证** | 试运行确认恢复正常 | 10% |

→ **诊断阶段是大数据最能发力的地方**：基于历史故障数据和传感器特征，可以缩短故障定位时间。

## 降低 MTTR 的策略

| 策略 | 方法 | 数据支撑 |
|------|------|----------|
| **快速诊断** | 故障知识库 + 诊断决策树 | 历史故障记录 |
| **备件预判** | 提前准备可能需要的备件 | [[MTBF-平均故障间隔时间|MTBF]] + 备件消耗趋势 |
| **远程支持** | AR 辅助维修 | 设备 3D 模型 + 维修手册 |
| **预防性维护** | 在故障前主动维护 | [[预测性维护|预测性维护]] 模型 |

## 与 [[MTBF-平均故障间隔时间|MTBF]] 的关系

$$故障可用率 = \frac{MTBF}{MTBF + MTTR}$$

- MTBF 决定了"多久坏一次"（可靠性）
- MTTR 决定了"坏了修多久"（维护性）
- 两者共同影响 [[可用率-Availability|可用率]]

## 数据采集要点

```
设备ID, 故障发生时间, 维修人员到场时间,
故障定位时间, 修复完成时间, 试运行通过时间,
故障类型代码, 更换备件清单, 修复措施描述
```
