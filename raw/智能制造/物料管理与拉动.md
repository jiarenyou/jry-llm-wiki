# 物料管理与拉动

## 概述

物料管理是智能制造中连接供应链与生产线的核心环节。物料成本通常占制造企业产品成本的50%-70%，物料管理水平直接影响库存周转、资金占用和交付效率。在精益生产理念下，物料管理的核心思想是"拉动"——由后工序根据实际需求向前工序提出物料需求，而非传统的"推动"式按计划大批量配送。

物料管理与[[生产计划与排产-APS]]、[[生产执行与报工]]紧密关联：APS提供物料需求计划，MES提供实时消耗数据，物料系统确保正确物料在正确时间到达正确位置。

## 推动与拉动

### 推动式生产

传统MRP驱动的方式：

- 基于预测和订单生成物料需求计划。
- 按计划批量生产、批量配送。
- 问题：库存积压、响应迟缓、牛鞭效应。

### 拉动式生产

精益思想驱动的方式：

- 由客户需求拉动最终装配，逐级向上游传递需求。
- 通过看板、电子信号等可视化工具传递需求信息。
- 优势：降低在制品、缩短交付周期、暴露问题。

### 推拉结合

实际运营中通常采用推拉结合策略：

- **推动层**：长周期原材料基于预测提前采购。
- **拉动层**：短线标准件和半成品基于看板拉动配送。
- **客户订单分离点（CODP）**：推动与拉动的切换节点，是供应链设计的关键决策点。

## 看板系统

看板（Kanban）是拉动系统的核心工具，是实现"准时化生产（JIT）"的信息载体。

### 看板类型

| 看板类型 | 功能 | 触发条件 |
|---------|------|---------|
| 生产看板 | 指示前工序生产什么、生产多少 | 后工序取走零件 |
| 取货看板 | 指示后工序去哪里取什么物料 | 消耗达到触发量 |
| 信号看板 | 批量生产触发信号 | 库存降至订货点 |
| 电子看板 | 数字化看板替代纸质看板 | 系统自动触发 |

### 看板数量计算

```
看板数量 = (日均需求 × 补货周期 × (1 + 安全系数)) / 容器容量
```

其中：
- **日均需求**：根据APS排产计划或历史消耗计算。
- **补货周期**：从触发到物料送达的总时间。
- **安全系数**：应对需求波动和供应延迟的缓冲，通常为0.1-0.3。
- **容器容量**：每个看板对应的标准化搬运数量。

### 电子看板系统

数字化时代的看板系统：

- **自动触发**：MES报工消耗自动触发物料需求，无需人工操作。
- **配送指令**：向AGV或物流人员推送取货-配送任务。
- **库存联动**：实时更新库位库存，自动计算补货需求。
- **异常报警**：缺料预警、超期未配、料不符时自动报警。

## 物料配送模式

### JIT准时配送

JIT配送的目标是"需要的物料、在需要的时间、按需要的数量送到需要的地点"：

- **频次配送**：按节拍时间（如每2小时）循环配送，减少线边库存。
- **成套配送**：按工单BOM配齐一套物料，整体配送到工位。
- **排序配送**：按生产序列排列物料配送顺序，适用于混线生产（如汽车总装）。

### 线边仓管理

线边仓是产线旁的物料暂存区域：

- **定置定位**：每个物料的存放位置固定并可视化标识。
- **最大最小量**：设定每个库位的库存上下限，触发自动补料。
- **FIFO管理**：先进先出，确保物料在保质期内使用。
- **盘点管理**：循环盘点和定期盘点结合，保持账实一致。

### AGV配送调度

智能物流场景下的AGV调度：

- **路径规划**：基于车间布局的最优路径计算，避障和交通管制。
- **多车协同**：多台AGV的任务分配和死锁避免。
- **充电调度**：在低峰时段自动安排充电，保障高峰运力。
- **与MES联动**：接收MES配送指令，配送完成自动回写状态。

## 库存管理

### ABC分类管理

| 分类 | 占比 | 管理策略 |
|------|------|---------|
| A类（高价值） | 品种占10-20%，金额占70-80% | 严密管控、小批量高频采购、精确预测 |
| B类（中价值） | 品种占20-30%，金额占15-20% | 适度管控、定期 review |
| C类（低价值） | 品种占50-70%，金额占5-10% | 简化管控、批量采购、高安全库存 |

### 安全库存设定

```
安全库存 = Z值 × 标准差 × √平均提前期
```

- **Z值**：服务水平对应的正态分布分位数（95%对应1.65，99%对应2.33）。
- **标准差**：需求波动的统计量。
- **动态调整**：基于历史消耗数据和服务水平目标定期更新。

### VMI供应商管理库存

VMI（Vendor Managed Inventory）是将库存管理责任转移给供应商的合作模式：

- **信息共享**：实时向供应商共享库存数据和消耗预测。
- **自动补货**：供应商根据约定的库存上下限自动补货。
- **结算方式**：按实际消耗定期结算，降低采购方资金占用。
- **适用场景**：标准件、通用件和长尾物料。

## WMS仓储管理系统

### 核心功能

```
入库管理 → 库位管理 → 库存管理 → 出库管理 → 批次追溯
    ↓          ↓          ↓          ↓          ↓
  收货验收   库位分配   盘点管理   拣货策略   正反向追溯
  上架规则   容量管理   库龄分析   波次释放   效期管理
```

### 入库流程

1. **收货预告**：接收ERP采购到货预告。
2. **到货登记**：核对送货单、采购单和实物。
3. **质量检验**：触发[[质量管控流程]]中的IQC检验。
4. **上架入库**：系统推荐最优库位，RFID或扫码确认上架。
5. **账务更新**：实时更新库存台账。

### 出库流程

1. **需求生成**：MES报工触发或计划自动生成出库需求。
2. **波次合并**：将多个出库需求合并为一个波次。
3. **拣货执行**：按系统指引的路径和顺序拣货。
4. **复核装箱**：扫码复核物料和数量。
5. **发货确认**：更新库存，同步MES和ERP。

## 物料追溯体系

### 追溯层级

- **批级追溯**：以原材料批次为单位追踪流向，适用于大宗原材料。
- **件级追溯**：以单个产品序列号追踪，适用于高价值或安全关键产品。
- **容器追溯**：以周转容器/托盘为单位追踪，适用于标准件配送。

### 追溯技术

| 技术 | 优势 | 局限 |
|------|------|------|
| 条码 | 成本低、实施简单 | 易污损、需逐个扫描 |
| RFID | 批量读取、无需可视 | 成本较高、金属环境干扰 |
| 二维码 | 信息容量大、可读性强 | 需逐个扫描 |
| 激光打标 | 永久性、耐高温 | 需专用设备 |

### 正反向追溯

- **正向追溯（Tracking）**：原材料批次 → 半成品 → 成品 → 客户，用于召回场景。
- **反向追溯（Tracing）**：客户投诉 → 成品批次 → 加工过程 → 原材料批次，用于根因分析。
- **追溯深度**：通常要求追溯到直接原材料供应商，关键产品需追溯到原材料原始批次。

## 数字化物料管理

### 物料主数据管理

物料主数据是所有物料管理活动的基础：

- **物料编码**：唯一标识，遵循企业编码规则。
- **物料描述**：标准化的命名和规格描述。
- **计量单位**：基本单位和采购单位的转换关系。
- **存储条件**：温湿度要求、保质期、危险品等级。
- **BOM关系**：与[[ERP-企业资源计划]]中产品结构的关联。

### 智能物料管理技术

- **AI需求预测**：基于历史消耗、订单预测和季节性因素预测物料需求。
- **智能库位推荐**：根据物料特性、出库频率和关联性推荐最优库位。
- **自动补货决策**：基于消耗速率、提前期和库存策略自动触发采购建议。
- [[数字孪生-Digital Twin]]在仓储物流中的应用：虚拟仓库仿真优化布局和动线。

## 实施路径

1. **基础整理阶段**：物料编码标准化、库位定置化、盘点常态化。
2. **系统化阶段**：部署WMS，实现入库-存储-出库的数字化管理。
3. **拉动化阶段**：建立看板系统，实现JIT配送和线边仓管理。
4. **智能化阶段**：引入AGV、RFID和AI预测，构建智能物流体系。

## 总结

物料管理与拉动的本质是在保障生产连续性的前提下，以最小的库存和最快的速度满足生产需求。从传统的推动式计划配送到精益拉动式JIT配送，再到智能化的自动补货和AI预测，物料管理经历了从经验驱动到数据驱动的深刻变革。与[[生产计划与排产-APS]]、[[生产执行与报工]]和[[质量管控流程]]的深度集成，将推动物料管理从"仓库保管"升级为"供应链协同中枢"，实现库存优化、成本降低和交付提速的统一目标。
