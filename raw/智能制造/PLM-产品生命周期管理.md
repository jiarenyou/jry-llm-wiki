# PLM — 产品生命周期管理

## 定义

PLM（Product Lifecycle Management，产品生命周期管理）是**管理产品从概念设计、研发、制造到退市全生命周期中所有数据与流程**的集成化平台。PLM 的核心使命是确保"正确的人在正确的时间使用正确的产品数据"。

在智能制造体系中，PLM 是**产品知识的单一数据源（Single Source of Truth）**——BOM、CAD 图纸、工艺规程、测试规范等全部在此管理。

## 产品全生命周期阶段

```
概念 → 设计 → 工程化 → 制造 → 服务 → 退市
 │       │       │        │       │       │
 需求    CAD    工艺BOM   ERP    备件    回收
 分析   仿真    工装夹具   MES    维修    处置
```

| 阶段 | PLM 管理内容 | 关键输出 |
|------|-------------|----------|
| **概念** | 市场需求、竞品分析 | 产品需求文档（PRD） |
| **设计** | CAD 模型、仿真数据 | 设计 BOM（EBOM） |
| **工程化** | 工艺路线、工装设计 | 工艺 BOM（MBOM）、SOP |
| **制造** | 生产数据、质量记录 | 制造执行包 |
| **服务** | 备件清单、维修手册 | 服务手册、备件 BOM |
| **退市** | 变更记录、合规文档 | 合规报告 |

## 核心功能模块

### 产品数据管理（PDM）

PDM 是 PLM 的数据基础，管理产品相关的所有文档和数据：

- **文档管理**：CAD 文件、Office 文档、PDF 的检入/检出/版本控制
- **BOM 管理**：从设计 BOM 到制造 BOM 的多视图管理
- **零部件管理**：标准件库、通用件库、物料分类编码
- **权限与流程**：数据访问权限、审批工作流

### 变更管理

工程变更（ECN/ECO）是制造业最复杂的流程之一：

1. **变更申请**（ECR）：描述问题、提出方案
2. **变更评估**：影响分析——影响哪些 BOM、在制品、库存
3. **变更执行**（ECO）：修改图纸、更新 BOM、通知相关方
4. **变更验证**：确认变更已生效

一个工程变更如果未及时同步到[[ERP-企业资源计划]]，可能导致采购错误物料、生产出不合格品。

### BOM 管理——连接设计与制造的纽带

BOM 是制造业最重要的数据结构之一：

| BOM 类型 | 来源 | 用途 |
|----------|------|------|
| **EBOM**（设计 BOM） | PLM / CAD | 反映产品设计结构 |
| **MBOM**（制造 BOM） | PLM / CAPP | 反映制造装配顺序 |
| **BBOM**（服务 BOM） | PLM / 售后 | 反映维修备件结构 |

PLM 负责 EBOM → MBOM 的转化过程，确保设计意图正确传递到制造环节。

## PLM 与其他系统的集成

### PLM → [[ERP-企业资源计划]]

- 传递内容：MBOM、工艺路线、标准工时、物料主数据
- 集成意义：确保 ERP 的 MRP 运算基于最新、最准确的产品数据
- 典型问题：PLM 中已更新图纸，但 ERP 中 BOM 未同步，导致采购和生产错误

### PLM → [[MES-制造执行系统]]

- 传递内容：工艺规程、SOP 文档、质检标准、工艺参数
- 集成意义：MES 执行的工艺必须是 PLM 中最新发布的版本

### PLM → [[QMS-质量管理系统]]

- 传递内容：检验规范、质量标准、不合格品处理流程
- 集成意义：质量问题闭环——从质量事件驱动设计改进

## 在[[智能制造概述与工业4.0]]中的定位

PLM 在工业 4.0 架构中承担**产品数字化主线（Digital Thread）**的角色：

- **数字孪生（Digital Twin）**：PLM 管理产品的三维模型和仿真数据，与[[SCADA-数据采集与监视]]的实时数据结合，构建产品的数字孪生体
- **跨学科协同**：机械、电子、软件三者的协同设计在 PLM 中统一管理
- **合规与可追溯**：汽车行业的 TS16949、航空的 AS9100 都要求完整的产品数据追溯链

## 实施关键

| 挑战 | 应对策略 |
|------|----------|
| 历史数据迁移 | 分阶段迁移，优先迁移活跃产品 |
| 标准件/通用件管理 | 建立企业标准件库，推进零部件标准化 |
| 与 ERP 的 BOM 同步 | 建立清晰的 BOM 所有权和变更通知机制 |
| 用户采纳率 | 从设计部门入手，逐步扩展到工艺和制造 |
