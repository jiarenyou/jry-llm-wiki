## Summary
详解工厂数据采集层的三大数据源分类（设备数据OT域、业务数据IT域、外部数据），设备数据采集链路（PLC/传感器 -> 协议网关 -> MQTT Broker -> Kafka -> 时序数据库/计算引擎），工业协议详解（OPC-UA、Modbus、MQTT、EtherNet/IP），边缘计算设计（数据清洗、协议转换、边缘推理），以及应对网络不稳定、协议异构、时钟同步和采集点位管理等核心挑战的策略。

## Key Claims
- 数据采集层是工厂数据平台的"第一公里"，直接决定平台的数据完整性和时效性
- EdgeX Foundry是工厂边缘计算的主流开源框架，支持插件式协议驱动管理
- OPC-UA适用于新设备接入和跨系统数据交换，是ISA-95 Level 2-3层数据交互的首选协议
- MQTT Topic设计推荐按"工厂/车间/产线/设备/数据类型"层级组织
- 边缘计算核心价值：数据清洗、协议转换和边缘推理，降低带宽需求和响应延迟
