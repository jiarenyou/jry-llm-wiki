# MTBF — 平均故障间隔时间

## 定义

MTBF（Mean Time Between Failures）衡量的是**设备两次故障之间的平均运行时间**。它是设备可靠性的核心指标——MTBF 越长，设备越可靠。

## 计算公式

$$MTBF = \frac{总运行时间}{故障次数}$$

## 示例

某设备一个月内运行了 600 小时，期间发生 3 次故障：
$$MTBF = \frac{600}{3} = 200 \text{ 小时}$$

含义：平均每运行 200 小时会出一次故障。

## 与 [[MTTR-平均修复时间|MTTR]] 的关系

MTBF 和 [[MTTR-平均修复时间|MTTR]] 共同决定设备的故障可用率：

$$故障可用率 = \frac{MTBF}{MTBF + MTTR}$$

| 情景 | MTBF | MTTR | 故障可用率 |
|------|------|------|-----------|
| 可靠但难修 | 500h | 10h | 98.0% |
| 不可靠但好修 | 100h | 2h | 98.0% |
| 又不可靠又难修 | 100h | 10h | 90.9% |

→ **提升 [[可用率-Availability|可用率]] 有两条路：降低故障频率（↑MTBF）或加快修复速度（↓MTTR）**

## 大数据应用

MTBF 是[[预测性维护|预测性维护]]的核心目标变量：

1. **历史分析**：按设备类型、产线、班次统计 MTBF 趋势
2. **异常检测**：MTBF 突然缩短 → 可能存在系统性问题
3. **预测模型**：基于振动、温度等传感器数据预测剩余寿命（RUL）
4. **维护优化**：根据 MTBF 制定预防性维护周期，避免过度维护或维护不足

## 数据采集要点

```
设备ID, 故障发生时间, 故障恢复时间,
故障类型代码, 故障部位, 维修记录,
设备运行时长累计
```

数据来源：SCADA 报警日志 + MES 维修工单 + CMMS 维护管理系统
