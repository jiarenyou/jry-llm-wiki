# 生产执行与报工

## 概述

生产执行与报工是[[MES-制造执行系统]]最核心的功能模块，负责将[[生产计划与排产-APS]]下发的排程方案转化为具体的车间作业指令，并实时采集生产过程数据形成闭环反馈。这一环节是制造企业实现"计划-执行-反馈"闭环管理的关键节点。

生产执行覆盖从工单下达到成品入库的完整过程，包括派工、领料、加工、检验、报工、转序和入库等环节。报工则是对执行过程的数字化记录，是后续成本核算、绩效考核和质量追溯的数据基础。

## 生产执行核心流程

### 工单管理与派工

工单是生产执行的基本单元，其生命周期包括：

1. **工单接收**：MES从[[ERP-企业资源计划]]或APS接收生产工单，包含产品型号、计划数量、交付日期等信息。
2. **工单拆分**：根据批量规则将大工单拆分为可执行的子批次，便于并行生产和在制品管理。
3. **工序派工**：按照[[PLM-产品生命周期管理]]定义的工艺路线，将工序分配到具体设备、班组和人员。
4. **资源校验**：派工前自动校验物料齐套性、设备可用性和人员资质。
5. **工单下达**：通过车间终端、移动PDA或电子看板将作业指令推送到操作人员。

### 作业指导推送

智能化的生产执行系统应实现：

- **SOP电子化**：将纸质作业指导书转化为电子文档，与工序绑定，按步骤推送。
- **工艺参数下发**：将配方、温度、压力等工艺参数直接下发到PLC或数控设备。
- **防错校验**：通过条码扫描、RFID或视觉识别确认物料、工装和操作的准确性。
- **强制检验点**：在关键工序设置质量门禁，未通过检验不允许流转下一工序。

### 物料配送执行

基于[[物料管理与拉动]]的配送指令：

- **JIT配送**：按工单开工时间倒推物料需求时点，触发AGV或人工配送。
- **线边仓管理**：管理产线旁的暂存物料，确保物料在正确时间到达正确位置。
- **缺料预警**：当物料低于安全线时自动触发补料请求。

## 报工体系设计

### 报工模式

| 模式 | 适用场景 | 数据粒度 |
|------|---------|---------|
| 按工单报工 | 简单装配、批量生产 | 工单级别汇总 |
| 按工序报工 | 多工序机加工 | 工序级别详细 |
| 按人员报工 | 计件工资、绩效核算 | 个人产出明细 |
| 自动报工 | 自动化产线、连线生产 | 设备直采实时数据 |

### 报工数据采集

**人工报工方式：**

- 车间触摸屏终端：操作员刷卡登录，选择工单和工序，录入产出数量。
- 移动PDA扫码报工：扫描工单条码快速定位，录入完工数据。
- 语音报工：适用于双手被占用的场景，通过语音指令完成报工。

**自动报工方式：**

- 设备直连采集：通过OPC UA、Modbus等协议从PLC/传感器实时采集产量、节拍数据。
- 视觉计数：通过工业相机识别成品下线数量。
- 称重换算：通过在线称重设备将重量换算为产出数量。

### 报工数据要素

一条完整的报工记录应包含：

```
工单号 | 工序号 | 操作员 | 设备编号 | 开始时间 | 结束时间
合格数量 | 不良数量 | 不良代码 | 废品数量 | 工时消耗 | 备注
```

这些数据是后续成本核算、质量分析和绩效评估的基础。

## 异常处理机制

### 生产异常分类

| 异常类型 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 设备异常 | 设备停机、故障报警 | 触发维修工单，通知维修人员 |
| 质量异常 | SPC超限、首检不合格 | 停线检查，启动不合格品处理流程 |
| 物料异常 | 缺料、来料不良 | 触发紧急配料或供应商反馈 |
| 人员异常 | 缺勤、技能不匹配 | 触发人员调配或培训计划 |
| 工艺异常 | 参数偏差、操作违规 | 触发工艺工程师介入 |

### Andon（安灯）系统

Andon系统是生产异常实时响应的核心工具：

- **异常呼叫**：操作员通过按钮、拉绳或终端触发Andon信号。
- **分级响应**：按异常严重程度设置不同响应级别（如黄色预警、红色停线）。
- **计时管理**：记录异常响应时间和处理时间，纳入绩效统计。
- **数据积累**：所有异常事件形成数据库，支持柏拉图分析和持续改善。

### 异常对排产的影响

当生产异常发生时，MES需要：

1. 评估异常影响范围和预计恢复时间。
2. 将异常信息实时反馈给[[生产计划与排产-APS]]。
3. 对于严重影响交付的异常，触发APS重新排产。
4. 对于局部异常，调整后续工序的开工时间和物料配送计划。

## 数据集成与可视化

### 与上游系统集成

```
ERP ──→ 工单/ BOM / 工艺路线 ──→ MES
APS ──→ 详细排程/ 派工指令 ──→ MES
PLM ──→ 工艺参数/ 质量标准 ──→ MES
```

### 与下游系统集成

```
MES ──→ 完工报告/ 工时数据 ──→ ERP（成本核算）
MES ──→ 质量数据/ 检验结果 ──→ QMS（质量系统）
MES ──→ 设备状态/ 运行参数 ──→ [[设备管理全流程]]（设备系统）
MES ──→ 物料消耗/ WIP数据 ──→ WMS（仓储系统）
```

### 车间可视化

- **电子看板**：实时展示工单进度、设备状态、产量达成率。
- **数字仪表盘**：SPC图表、OEE趋势、异常柏拉图。
- **进度追踪**：工单在制状态追踪，支持甘特图和流程图视图。

## 移动化与无纸化

现代MES的生产执行模块正向移动化和无纸化方向演进：

- **电子签批**：替代纸质流转卡，通过电子签名实现工序签收。
- **移动巡检**：管理人员通过Pad巡检车间，实时查看产线状态。
- **AR辅助**：通过AR眼镜叠加作业指导、物料位置和设备状态信息。
- **区块链追溯**：将关键报工数据上链，确保数据的不可篡改性。

## 实施要点

1. **报工粒度选择**：在数据精度和操作成本之间取得平衡，并非越细越好。
2. **用户体验设计**：报工界面要简洁直观，操作步骤不超过3步。
3. **离线容错**：车间网络不稳定时支持离线报工，恢复后自动同步。
4. **数据校验**：通过规则引擎实时校验报工数据的合理性（如产出不能超过计划量×损耗率上限）。
5. **培训与变革**：一线操作员的接受度是系统成功的关键，需充分培训和试运行。

## 总结

生产执行与报工是MES系统的"心脏"，其核心价值在于将计划转化为行动、将经验转化为数据。一个设计良好的生产执行系统，应当做到"数据不落地"——从工单下达到完工入库的全过程实现数字化流转。与[[生产计划与排产-APS]]、[[质量管控流程]]和[[物料管理与拉动]]的深度集成，将推动制造企业从"事后统计"走向"事中控制"和"事前预测"。
