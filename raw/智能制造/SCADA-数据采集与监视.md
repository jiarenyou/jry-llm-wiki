# SCADA — 数据采集与监视

## 定义

SCADA（Supervisory Control And Data Acquisition，数据采集与监视控制系统）是工业自动化中用于**实时采集现场设备数据并进行集中监视与控制**的系统。在[[ISA-95参考架构]]中，SCADA 位于**第1-2层（Level 1-2）——感知与控制层**，是连接物理设备与上层管理系统的桥梁。

如果说[[ERP-企业资源计划]]是企业的"大脑"、[[MES-制造执行系统]]是"脊髓"，那么 SCADA 就是遍布工厂的"神经网络"——感知设备的每一个状态变化。

## 系统架构

SCADA 系统通常采用分层分布式架构：

```
┌─────────────────────────────────┐
│       SCADA Server（主站）       │  数据汇聚、告警处理、历史存储
├─────────────────────────────────┤
│    Communication Network        │  工业以太网 / 无线 / 串口
├─────────────────────────────────┤
│  RTU / PLC / RTU（现场控制器）    │  本地逻辑控制、协议转换
├─────────────────────────────────┤
│  Sensors & Actuators（现场设备） │  温度、压力、流量、电机、阀门
└─────────────────────────────────┘
```

### 核心组件

| 组件 | 功能 | 典型产品 |
|------|------|----------|
| **RTU**（远程终端单元） | 数据采集、协议转换、简单逻辑 | ABB、Schneider |
| **PLC**（可编程逻辑控制器） | 实时逻辑控制、顺序控制 | Siemens S7、Mitsubishi |
| **HMI**（人机界面） | 现场操作员监控界面 | 触摸屏、操作员站 |
| **SCADA Server** | 数据汇聚、历史数据库、报警引擎 | WinCC、iFIX、Ignition |
| **通信网络** | 现场总线 / 工业以太网 | Profinet、Modbus TCP、OPC UA |

## 关键通信协议

| 协议 | 特点 | 应用场景 |
|------|------|----------|
| **Modbus TCP/RTU** | 简单、开放、古老但广泛使用 | 通用设备通信 |
| **OPC UA** | 安全、跨平台、信息模型丰富 | 工业4.0标准协议 |
| **Profinet** | 实时性好、Siemens 生态 | 过程控制、运动控制 |
| **EtherNet/IP** | Allen-Bradley 生态、CIP 协议 | 北美市场主流 |
| **MQTT** | 轻量级、发布/订阅模式 | IoT 场景、远程监控 |

OPC UA 正在成为[[智能制造概述与工业4.0]]中设备互联的标准协议，其信息模型可以完整描述设备能力和数据语义。

## SCADA 与 MES 的数据接口

SCADA 是[[MES-制造执行系统]]最重要的底层数据来源：

| 数据类型 | 采集频率 | MES 用途 |
|----------|----------|----------|
| 设备运行状态 | 1-5 秒 | 计算[[可用率-Availability]]、设备利用率 |
| 实时产量（计数器） | 1-10 秒 | 生产进度跟踪、[[性能率-Performance]]计算 |
| 工艺参数（温度/压力等） | 1-60 秒 | 过程质量控制、SPC 分析 |
| 报警记录 | 事件触发 | 故障分析、[[MTBF-平均故障间隔时间]]计算 |
| 能耗数据 | 1-60 秒 | 能效分析、碳排放追踪 |

## 核心功能

### 实时监控

SCADA 的核心能力是**毫秒级**的数据采集和画面刷新。操作员可以在中央控制室看到整个工厂每台设备的运行状态——运行、停机、故障、待机。

### 报警管理

报警系统是 SCADA 最重要的安全功能之一：

- **报警分级**：紧急、高、中、低四个等级
- **报警抑制**：避免"报警洪泛"（Alarm Flood）
- **报警归档**：历史报警记录用于故障追溯

有效的报警管理直接关系到[[OEE-设备综合效率]]的提升——减少无效报警、加快故障响应。

### 历史数据存储

SCADA 系统通常配置历史数据库（Historian），存储高频率的时序数据。这些数据是上层数据分析和机器学习的基础。

### 远程控制

SCADA 不仅"看"，还能"做"——远程启停设备、修改设定值、切换工艺配方。控制权限管理是安全设计的核心。

## 行业应用

| 行业 | 监控对象 | 特点 |
|------|----------|------|
| **石油天然气** | 管线、油井、储罐 | 分布范围广、远程监控 |
| **电力** | 变电站、配电网 | 高可靠性要求 |
| **水处理** | 泵站、水质 | 连续过程、环境合规 |
| **制造业** | 产线设备、PLC | 与[[MES-制造执行系统]]紧密集成 |

## 安全挑战

SCADA 系统的安全是工业信息安全的核心：

- **网络安全**：IT/OT 融合带来网络攻击风险（如 Stuxnet 震网病毒）
- **访问控制**：分级授权、操作审计
- **纵深防御**：DMZ 隔离、防火墙、VPN
- **安全标准**：IEC 62443 工控安全标准

## 演进趋势

SCADA 正在从传统的封闭系统向**开放、智能、云端协同**的方向演进：

- **边缘计算**：在设备端部署 AI 模型，实现毫秒级的预测性维护
- **云 SCADA**：基于云平台的远程监控，适合分布式设备管理
- **数字孪生**：SCADA 实时数据驱动工厂三维模型，实现虚实同步
- **与[[CMMS-设备维护管理]]联动**：设备异常自动触发工单，缩短[[MTTR-平均修复时间]]
